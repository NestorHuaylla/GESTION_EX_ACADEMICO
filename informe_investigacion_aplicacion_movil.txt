\section{Datos generales}

\subsection{Título del proyecto}
\begin{itemize}
    \item Aplicación móvil para la gestión y visualización de expedientes académicos mediante Árbol Binario de Búsqueda.
\end{itemize}

\subsection{Síntesis de documentos revisados}

Se revisaron los archivos disponibles en la carpeta del proyecto. La esencia común de los documentos es el diseño de un sistema académico que organiza expedientes estudiantiles por código mediante un Árbol Binario de Búsqueda (BST), permitiendo insertar, buscar, eliminar, recorrer, filtrar y analizar registros académicos.

\section{Idea de la aplicación}

\subsection{Nombre de la aplicación}

BST Académico Móvil.

\subsection{Problema que resuelve}

En una institución académica, la consulta de expedientes de estudiantes puede volverse lenta si los registros se revisan manualmente o si no existe una estructura clara para localizar datos por código. El proyecto propone una aplicación móvil que permita registrar, buscar y consultar expedientes académicos usando un Árbol Binario de Búsqueda, estructura que ordena los registros por código de estudiante y facilita búsquedas, recorridos y filtros.

La propuesta parte de los documentos revisados, donde los expedientes contienen código, nombre, escuela profesional, PPA, créditos, estado académico y semestre de ingreso. En la prueba ejecutada con la versión Python se trabajó con 7 estudiantes, altura de árbol 2, PPA promedio de 15.37, 5 estudiantes activos y operaciones correctas de búsqueda, filtrado y eliminación.

\subsection{Usuarios objetivo}

\begin{itemize}
    \item Personal de secretaría académica que necesita ubicar expedientes por código de estudiante.
    \item Docentes o tutores que requieren revisar estado académico, PPA, créditos y escuela profesional.
    \item Coordinadores de programa que necesitan filtrar estudiantes por estado, escuela o rendimiento.
    \item Estudiantes que podrían consultar su información académica básica desde el móvil, si se incorpora autenticación.
\end{itemize}

\subsection{Funcionalidades principales}

\begin{itemize}
    \item Registro de expedientes académicos con validación de código, PPA, créditos, estado y semestre.
    \item Búsqueda de estudiante por código usando la lógica del BST.
    \item Inserción y eliminación de registros manteniendo el orden del árbol.
    \item Recorridos In-Order, Pre-Order, Post-Order y BFS por niveles.
    \item Filtros por escuela profesional, estado académico, rango de PPA y rango de códigos.
    \item Visualización gráfica del árbol para comprender la ubicación de cada expediente.
    \item Panel de estadísticas: total de nodos, altura del árbol, PPA mínimo, PPA máximo, PPA promedio y estudiantes activos.
    \item Comparación de rendimiento entre implementaciones o módulos internos para evaluar la viabilidad técnica.
\end{itemize}

\section{Conclusiones}

\begin{itemize}
    \item La esencia del proyecto es usar un Árbol Binario de Búsqueda para organizar expedientes académicos por código de estudiante y facilitar consultas ordenadas.
    \item Los documentos revisados muestran una base funcional sólida: la lógica existe en Python, C++ y JavaScript, y el simulador web demuestra visualización, recorridos, filtros y comparación de rendimiento.
    \item La aplicación móvil es viable porque las funciones principales ya están definidas: búsqueda, inserción, eliminación, recorridos, filtros y estadísticas académicas.
    \item La principal mejora para una versión móvil real sería agregar autenticación, persistencia en base de datos, sincronización con backend y protección de datos personales.
    \item Las aplicaciones similares investigadas demuestran que existe necesidad de acceso móvil a información académica, pero la propuesta se diferencia porque incluye visualización del BST y explicación algorítmica del expediente.
\end{itemize}
